# Ubiquitous Language

This document is the authoritative glossary for the SaveStateGG domain. Every term defined here **must** be used consistently across all codebases, APIs, databases, and team communications. When a term is ambiguous or its meaning evolves, this document should be updated first and all impacted repositories should align accordingly.

> **Last updated:** 2026-03-31

## Core Domain Terms

| Term | Definition | Notes |
|---|---|---|
| **Game** | A video game title that exists in the SaveStateGG catalog. Identified by a unique slug and associated with metadata such as release date, developer, publisher, genres, and platform(s). | The canonical source for Game data is the platform catalog. Games are not created by end-users directly. |
| **Log** | A record that a user has interacted with a Game in a specific way (e.g., played, playing, want to play, dropped, completed). A Log captures *when* and *how* a user engaged with a Game, without necessarily including a written opinion. | A user may have at most one Log per Game. Updating a Log replaces the previous entry for that Game. |
| **Review** | A written evaluation of a Game authored by a user. A Review may optionally include a numeric rating. A Review is always associated with exactly one Game and one User. | A Review is distinct from a Log; a user may Log a Game without writing a Review. A Review implicitly creates or updates a Log. |
| **List** | A user-curated, ordered or unordered collection of Games, with an optional name, description, and visibility setting (public or private). | A Game may appear in multiple Lists. Lists are owned by a single User. |

## Supporting Terms

| Term | Definition | Notes |
|---|---|---|
| **User** | A registered account on the SaveStateGG platform. Users can log games, write reviews, and manage lists. | Authentication and identity management are handled separately from the core domain. |
| **Rating** | A numeric score (e.g., 0.5 – 5.0, in 0.5 increments) optionally attached to a Review to express the user's overall assessment of a Game. | Not all Reviews require a Rating. The rating scale may be adjusted in future iterations; any such change constitutes a breaking change to the domain and must be accompanied by a new ADR. |
| **Catalog** | The aggregate of all Games available on the platform. The Catalog is the bounded context responsible for Game discovery and metadata management. | External data sources (e.g., IGDB, RAWG) may be used to seed Catalog data, but the Catalog is the single internal source of truth for Game metadata. |
| **Activity Feed** | A chronological stream of public domain events (Logs, Reviews, List updates) generated by users that a given User follows. | The Activity Feed is a read model derived from domain events; it does not store primary data. |
